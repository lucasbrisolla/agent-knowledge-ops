# Method Wiki Framework

Define como conhecimento promovido entra na camada metodológica de um agente.

## Papel Do Method Wiki

O `method-wiki` guarda como o agente deve pensar, diagnosticar, decidir e revisar.

Ele não deve virar biblioteca de fontes brutas.

## Taxonomia Recomendada

| Tipo | Papel |
|---|---|
| `concepts/` | vocabulário e fundamentos estáveis |
| `principles/` | regras de julgamento de alto nível |
| `heuristics/` | atalhos de decisão e sinais recorrentes |
| `warnings/` | erros, vieses e anti-padrões |
| `questions/` | perguntas de challenge e descoberta |
| `checklists/` | revisões práticas |
| `patterns/` | estruturas recorrentes de análise |
| `processes/` | hubs metodológicos por processo |

## Regra Editorial

Cada nota deve ser utilizável sem consultar a fonte original.

Se a nota só resume um capítulo, ainda é destilação, não method wiki.

## Teste De Entrada

Antes de criar ou atualizar uma nota, perguntar:

- isso muda como o agente julga?
- isso reduz erro recorrente?
- isso melhora um workflow existente?
- isso deveria enriquecer arquivo existente?
- isso cria uma lacuna real que merece arquivo novo?
